## Lexique

Quelques termes qui reviennent souvent dans le dossier :

- **Deck** — une présentation slides (Canva, PowerPoint, Google Slides…).
  Dans notre cas : les 15 slides projetées le 27 mai.
- **Candidat (Canva)** — proposition de design générée par l'IA de Canva.
  Un candidat n'est **pas encore** un vrai fichier dans ton compte —
  c'est un brouillon qu'on peut prévisualiser et choisir.
- **Matérialiser un candidat** — convertir un brouillon Canva AI en
  vrai fichier Canva éditable (dans ton compte, dans le bon dossier).
  Action effectuée par l'admin via son agent Claude Code + MCP Canva.
- **Saynète** — la mise en scène d'ouverture du pitch (30 secondes,
  deux personnes, scène vécue réelle) qui accroche le jury avant les
  slides.
- **S0, S1, …, S14** — numérotation des 15 slides du deck. S0 = slide
  de garde de la saynète, S1-S14 = contenu du pitch.
- **USP** — *Unique Selling Proposition*, l'atout différenciant qu'on
  met en avant. Ici : le scanner code-barre.
- **Oral blanc** — pré-présentation à blanc devant le coach Ydays,
  prévue le 2026-05-06, pour ajuster avant la vraie soutenance du
  2026-05-27.
- **Board** — conseil stratégique envisagé côté gouvernance (cf.
  web-studio brainstorming).
- **MCP (Canva)** — *Model Context Protocol*, c'est le pont technique
  qui permet à l'agent Claude Code d'agir directement sur Canva
  (créer un dossier, générer un deck, matérialiser un candidat…).
