# Historique et décisions

Cette section sert à comprendre **pourquoi** certaines décisions ont
été prises. Elle n'est pas nécessaire pour une première lecture.

::: info Debriefs = snapshots datés, pas maintenus rétrospectivement
Chaque debrief documente l'**état du projet à sa date** (décisions,
hypothèses, références issues en cours à ce moment-là). Ils ne sont
**pas réécrits** quand la situation évolue — c'est ce qui permet de
retracer fidèlement les arbitrages.

Si un debrief cite des éléments qui ont depuis été revus (par exemple
des références d'issues abandonnées ou un format remplacé), ça
reflète la **réalité historique** et non une erreur à corriger. La
vue courante de référence est la [checklist de
statut](/outputs/soutenance-27-mai-status-checklist).
:::

## Quand lire cette section

- si une décision te paraît étrange ;
- si tu veux retracer les arbitrages ;
- si tu veux distinguer ce qui a été abandonné de ce qui est encore actif.

## Quand l'éviter

- si tu découvres le dossier ;
- si tu veux seulement relire le pitch ;
- si tu cherches les tâches restantes.
